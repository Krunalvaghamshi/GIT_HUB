The Development Journey

Phase 1: Scoping

Requirement Gathering

The project began with a clear business question: "Why are we losing customers?". Stakeholders needed a consolidated view of Churn, Usage (Call/SMS/Data), and Demographics across 4 major players: Jio, Airtel, Vodafone, and BSNL.

Input: telecom_churn.csv (243k rows)
Phase 2: Data Engineering (ETL)

Cleaning & Feature Engineering

Profiling revealed a critical bug: Negative Usage Values (e.g., Data = -361 MB). I engineered a Python/Excel pipeline to:

  • Convert negative usage to Absolute Values.
  • Create One-Hot Encoded flags (`Jio_Churned`, `Airtel_Churned`) to optimize Power BI performance.
df['Data_Clean'] = df['Data'].abs()
df['Jio_Flag'] = (df['Partner']=='Jio').astype(int)
Phase 3: Advanced Logic (DAX)

Statistical Modeling

To find the "Most Frequent Age" (Mode), standard aggregations failed. I wrote a complex DAX pattern using virtual tables (`SUMMARIZE`) to group users by age, count them, and extract the peak value dynamically for each provider.

MAXX(SUMMARIZE(Table, [Age], "Count", ...), [Count])
Phase 4: UI/UX Design

Dark Mode Architecture

Designed a high-contrast "App-Like" interface. Features include:

  • Persistent Navigation: Sidebar for instant switching between providers.
  • Bookmark Actions: A "RESET" button to clear complex slicers instantly.
  • Branding: Custom assets for each competitor page.
Output: 5 Interactive Report Pages
Phase 5: Executive Insights

The Findings

The final analysis revealed a critical risk: Vodafone's aging user base (Mode Age: 73) vs competitors (Age: 48). It also confirmed a systemic industry churn floor of ~20%, proving that customers are highly price-sensitive.

Result: Strategic Pivot Recommended

Report Audit Conclusion

Analyzing the static PDF export confirms the integrity of the Power BI dashboard data. The report explicitly details the "Month Wise" usage peaks (July/August) shown in the line charts and validates the provider-specific churn counts. The breakdown proves that churn is not an isolated incident but a market-wide systemic failure to retain users beyond the 20% threshold.
