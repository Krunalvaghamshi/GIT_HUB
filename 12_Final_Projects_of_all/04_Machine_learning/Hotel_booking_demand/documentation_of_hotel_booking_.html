The Development Lifecycle

Phase 01

Problem Identification & Data Acquisition

Hotels suffer revenue bleed from last-minute cancellations. The goal: Build an AI system to predict probabilities and trigger preemptive deposits.

  • Source: hotel_bookings.csv
  • Volume: 119,390 Observations
  • Target: is_canceled (Binary)

Phase 02

Exploratory Data Analysis (EDA)

Before coding the app, we used Jupyter Notebooks to uncover the logic behind booking behaviors.

# Insight Generation
Lead Time: High correlation with cancellation.
Segment: "Online TA" most volatile.
Deposit: Non-Refund = Revenue Secured.

Data Cleaning Strategy

  • Handled missing values in agent/company columns.
  • Filtered anomalies (0 adults/children bookings).
  • Identified key "Smart Drivers" for the app.
Phase 03

Preprocessing & Feature Engineering

Machine Learning requires math, not text. We built a pipeline to translate human concepts into model-ready vectors.

1. Encoding (The Critical Link)

Mapped categorical variables to integers. These mappings were hardcoded into app.py to ensure production consistency.

  • Resort Hotel → 0, City Hotel → 1
  • Online TA → 6, Offline TA → 5

2. Transformation

Applied Log Transformation to normalize skewed data distributions.

np.log1p(lead_time)
np.log1p(adr)
Phase 04

Model Training & Serialization

Algorithm Selection

Tested Logistic Regression & Random Forest.
Winner: XGBoost Classifier for speed and accuracy on tabular data.

  'model': trained_xgboost,

Phase 05

Application Development (Frontend)

Transitioning from Research (Notebook) to Production (Streamlit). We built a "Smart Input" system.

The Logic Core

  • Input: Date Range (Nov 1 - Nov 10)
  • Auto-Calc: Lead Time (9 days), Week (45)
  • Prediction: `model.predict_proba`

The Concierge Intelligence

  • > 70%: Critical Retention Action
  • High ADR: VIP Protocol Trigger
  • UI: Custom CSS (Midnight Blue & Gold)
Phase 06 & 07

Robustness & Deployment

Fail-Safe Systems

Cloud paths can be unpredictable. We implemented `find_file_in_dir` to recursively hunt for the model file, plus a manual admin upload fallback.

Architecture

  • User Input → Derived Metrics
  • Preprocessor → Encoding
  • XGBoost → Risk Score (0-100%)
  • Strategy Engine → Business Advice
