The Development Journey

From raw CSV files to a production-grade Streamlit application.

PHASE 1

The Foundation: Data Discovery

Starting with green_taxi_2013.csv, we decoded cryptic IDs (RateCodeID: 2 = JFK) and ran automated audits to visualize distribution skews before writing a single line of model code.

PHASE 2

The Science: EDA & Modeling

Feature Engineering was the secret sauce. We extracted time metrics and interaction features. After testing Linear Regression and Decision Trees, XGBoost emerged as the champion with 97% accuracy.

PHASE 3

The Architecture

Building the bridge between user input and the model matrix. We implemented the Haversine formula for distance, traffic logic simulators, and TLC surcharge rules to create a smart logic layer.

PHASE 4

UX Evolution (v7.0)

Moved to a Premium Dark Mode. Integrated PyDeck for 3D Arc Layers. Replaced Lat/Lon inputs with landmark selectors and added a "Smart Insight Engine" to explain pricing context.

Under The Hood

XGBoost Engine

Serialized into a `.pkl` artifact, this model handles complex non-linear relationships between traffic, time, and location better than linear regression.

PyDeck 3D

Visual rendering of the trip using Arc Layers. It visually connects pickup and drop-off points with a glowing path over a pitch-black map.

Streamlit Cloud

Deployed on Streamlit Cloud for instant accessibility. The app bridges Python logic with an interactive web frontend seamlessly.
