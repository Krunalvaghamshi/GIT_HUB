Beyond Keywords. Pure Meaning.

Most recommendation systems match "Action" to "Action". CineMind AI is different. It constructs a semantic profile of the user based on mood, era, and language preferences, converting these into high-dimensional vector embeddings to find movies that match the feeling, not just the tag.

THE SECRET SAUCE

Persistence in a Stateless World

The biggest challenge in deploying free Python apps is the "Amnesia Effect". Cloud containers (like Streamlit Cloud) are ephemeral—they destroy local files (`users.csv`) every time they restart.

The Rare Solution: instead of paying for a SQL database, we engineered a custom connection to the Google Sheets API. This effectively turns a spreadsheet into a real-time, cloud-native NoSQL database that persists forever and is free to use.

  • GCP Service Account: A "Robot" user with Editor access.
  • Streamlit Secrets: Injecting JSON credentials securely at runtime via TOML.
  • gspread Library: Python wrapper for read/write operations.

The Development Lifecycle

PHASE 01

The Foundation (CLI)

Built the core logic using sentence-transformers (all-MiniLM-L6-v2) and Cosine Similarity. The initial prototype ran entirely in the terminal using input() loops and local CSV storage.

PHASE 02

Interface Transformation

Migrated from CLI to Streamlit. Replaced while loops with Session State management to handle user flow without refreshing variables. Added progress bars for AI confidence scores.

PHASE 03

The "Amnesia" Crisis

Upon cloud deployment, I encountered the "Ephemeral File System" issue. Local CSVs were wiped every time the app slept or rebooted, causing total data loss for users.

PHASE 04

Cloud Persistence Architecture

Major Pivot: Engineered a connection to Google Sheets via GCP Service Accounts. Configured TOML secrets management to inject sensitive API keys securely at runtime, solving the data loss problem permanently.

PHASE 05

Intelligence & UX Refinement

Solved the "Infinite Recommendation Loop" using Session Exclusion Sets. Added rich metadata fetching (Directors, Cast, Trailers) and a "Vision-Full" UI with dark mode and movie posters.

Engineering Challenges Solved

Ephemeral Data Loss

Problem: Cloud containers destroy local files on restart.

Solution: Integrated gspread and Google Cloud Platform (GCP) Service Accounts to externalize the database layer, ensuring permanent history storage.

Security & Secrets

Problem: Cannot push API keys or JSON credentials to GitHub public repos.

Solution: Utilized TOML-based Streamlit Secrets injection for secure, environment-variable style credential management in production.

The Loop Bug

Problem: AI recommending skipped movies repeatedly.

Solution: Implemented a real-time skipped_ids session set that unions with the permanent seen_ids database to dynamically filter the recommendation pool.

Project Summary

CineMind AI represents a complete transformation from a local Python script to a resilient, cloud-native application. By leveraging Semantic Search (Hugging Face) for intelligence and Google Sheets API for persistence, it solves the "stateless container" problem that plagues most hobbyist deployments. The result is a robust, data-driven recommendation engine that learns from user behavior without incurring expensive database costs.
