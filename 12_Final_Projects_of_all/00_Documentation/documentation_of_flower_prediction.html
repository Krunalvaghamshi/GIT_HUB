Computational Botany

An end-to-end journey from raw pixel data to a production-grade AI system. Leveraging Transfer Learning and MobileNetV2 to democratize botanical knowledge.

The Challenge

Botanical classification in the wild is plagued by variable lighting, occlusion, and background noise. Training custom CNNs from scratch often leads to overfitting and low accuracy (capped at ~65%) due to limited dataset size.

‚ö°

The Solution

Adopted a Transfer Learning approach using MobileNetV2. By leveraging weights pre-trained on 1.4 million ImageNet samples, the model extracts complex features instantly, achieving high accuracy with minimal computational cost.

üåø

The Impact

Deployed as FloraMind, a sophisticated Streamlit web app. It serves as a digital field guide, offering real-time classification, taxonomy details, symbolism, and scientific uncertainty metrics for 5 distinct floral species.

01

Inception & Data CurationFoundation Layer

  • Dataset: Curated 4,300+ images across 5 classes (Daisy, Dandelion, Rose, Sunflower, Tulip).
  • Preprocessing: Implemented standardization pipeline resizing inputs to 256x256.
  • Normalization: Scaled pixel values to [0, 1] range for numerical stability.
OpenCV NumPy Data Augmentation
02

Custom ArchitectureThe Scratch Approach

  • Built a Sequential CNN from scratch to understand feature extraction.
  • Layers: Conv2D (Features), MaxPooling2D (Downsampling), Flatten, Dense.
  • Bottleneck: Encountered significant overfitting with capped validation accuracy (~65%).
TensorFlow Keras Sequential Conv2D
03

Hyperparameter TuningOptimization

  • Implemented Keras Tuner to automate the search for optimal architecture.
  • Algorithm: Used Hyperband to dynamically adjust learning rates and filter counts.
  • Result: Improved model stability but highlighted the need for deeper, pre-trained feature extractors.
Keras Tuner Hyperband Adam Optimizer
04

Transfer Learning PivotMobileNetV2 Integration

  • Leveraged MobileNetV2 pre-trained on ImageNet (1.4M images).
  • Input Shift: Adjusted tensors to 224x224 with [-1, 1] normalization.
  • Head Architecture: GlobalAveragePooling2D → Dense(Softmax).
  • Outcome: Achieved high accuracy with frozen base weights. Saved as .keras artifact.
MobileNetV2 Transfer Learning Feature Extraction
05

FloraMind InterfaceProduct Development

  • Developed a "Vision Full" UI using Streamlit with Glassmorphism aesthetic.
  • Smart Logic: Integrated BOTANICAL_DB for scientific context (Taxonomy, Symbolism).
  • Analytics: Added Entropy (Uncertainty) metrics via SciPy and Interactive Radar Charts via Plotly.
Streamlit Plotly.js SciPy Stats

Deployment & RobustnessFinal Polish

  • Ensured cross-platform compatibility via requirements.txt.
